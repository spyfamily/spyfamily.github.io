一、SSH config参数层次

1.命令行参数：

下面的例子就是命令行参数，-i ~/.ssh/spy是指使用指定的私钥文件，私钥文件地址~/.ssh/spy,-p 5022是指定端口SSH连接端口为5022，最后的git@gitee.com是用户名和连接地址。

2.用户配置文件

路径为~/.ssh/config，默认不存在，需要自行创建

3.全局配置文件

路径为/ext/ssh/ssh_config/

参数重要性：序号越小参数越重要，会被优先执行。

二、SSH config示例

从上述例子中可以得到：

  1. 配置写法有两种：参数名 参数值和参数名=参数值，并且两者可以混用
  2. 参数名不分大小写，参数值分大小写。例如：参数名HostName和Hostname等同
  3. 一个配置文件可以有多个Host
  4. #是注释符号

三、常见参数类型与详解

常见参数类型有：Host、HostName、User、IdentityFile、Port等

Host、HostName

因为两者密不可分，所以联合一起解释

  1. 在configuration 1配置文件下，仅仅为网址gitee.com，并配置私钥文件~/.ssh/spygitee，使用的时候输入命令ssh git@gitee.com即可连接成功。
  2. 在configuration 2配置文件下，除了为网址gitee.com配置私钥文件~/.ssh/spygitee外，还为这条配置取名(Host)为spy.gitee。因此在这种情况下使用命令ssh git@gitee.com会报错git@gitee.com: Permission denied (publickey).。而我们使用取的名称(Host)来实行命令，即ssh git@spy.gitee即可连接成功.

    在上述例子中，可以看到ssh命令是根据配置文件中的Host进行检测和匹配的。在例2中spy.gitee不是网址却连接成功的主要原因就是Hostname指向了网址gitee.com。同样在例2中，仅仅为spy。gitee指向的gitee.com配置的私钥文件，而直接指向gitee.com并不会匹配到相应配置，即无法匹配到相应私钥文件，所以会出现错误git@gitee.com: Permission denied (publickey).。

User

ssh连接的用户名，通常为@前的字符串。例如git@gitee.com的用户名就是git。若配置完用户名，将命令ssh git@gitee.com简写为ssh gitee.com。

IdentityFile

指定私钥证书，默认位置是~/.ssh/id_rsa，如果采用默认的证书，可以不用设置此参数，除非证书需要自定义，或与其他证书进行区分，就需要设置该参数来指向证书

Port

SSH访问主机的端口号，默认是22端口，同上，只有在非默认情况下才需要设置该值

其他

貌似常用的参数就这些，别的参数可以在命令行通过man ssh_config来查看，其实涉及的参数还是非常多的。已整理到下拉框，长度近千行，慎点。

man ssh_config

如果对三、常见参数类型与详解中的Host、Hostname有所了解后，实现Gitee的多账户配置自然也就不是难题。直接上例子。

需要注意的是我们从仓库拷贝的SSH地址，例如git@gitee.com:spy/spy.git,在:前的git@gitee.com需要替换为我们的Host，即spy.gitee:spy/spy.git。若未配置User参数则需要加上用户名spy.gitee:spy/spy.git
